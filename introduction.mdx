InQL is an open-source Burp Suite extension built for security professionals who test GraphQL APIs. It automates the reconnaissance work — introspecting endpoints, generating every possible query and mutation, detecting vulnerable patterns — so you can spend your time on actual findings rather than manual schema exploration. InQL v6.1.2 works with both Burp Suite Professional and Community editions and requires Java 17 or later.

## Who InQL is for

InQL is designed for penetration testers and bug bounty hunters who regularly encounter GraphQL APIs. If you've ever manually built GraphQL introspection queries, copied schema fields into requests one by one, or tried to figure out what operations an API exposes without documentation, InQL automates all of that inside Burp.

## Scanner capabilities

The **Scanner** tab is the core of InQL. Beyond basic introspection and query generation, it provides:

- **Points of Interest scanning** — surface fields related to authentication, PII, payments, file operations, debugging endpoints, and deprecated types
- **Circular reference detection** — identify recursive query structures that could be exploited for denial-of-service attacks
- **Custom headers** — set per-domain request headers, with the domain list auto-populated from Burp proxy traffic
- **Configurable depth and indentation** — control how deeply InQL generates nested queries and how output is formatted

## Burp integration

InQL registers a **GraphQL (InQL)** tab on Burp's native HTTP message editors, giving you syntax highlighting and formatting for any GraphQL request in Proxy, Repeater, or other Burp tools. You can send any auto-generated query directly to Repeater, Intruder, or other Burp tools via the context menu.

InQL also embeds **GraphiQL** and **GraphQL Voyager** servers, which you can use to explore an analyzed schema interactively.
