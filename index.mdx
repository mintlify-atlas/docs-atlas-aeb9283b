## What InQL does

InQL is built for security professionals who test GraphQL APIs. It handles the tedious parts of GraphQL reconnaissance — introspection, query generation, schema analysis — so you can focus on finding vulnerabilities.

## Key capabilities

<Steps>
  <Step title="Introspect and analyze">
    Point InQL at any GraphQL endpoint or load a local schema file. InQL automatically runs introspection and builds a complete map of all queries, mutations, and subscriptions.
  </Step>
  <Step title="Detect vulnerabilities">
    Run a Points of Interest scan to surface fields related to authentication, PII, payments, file operations, and more. Enable circular reference detection to find potentially dangerous recursive queries.
  </Step>
  <Step title="Generate and send">
    Send any auto-generated query directly to Burp Repeater, Intruder, or other tools. Use the built-in GraphQL editor tab on any request for syntax highlighting and formatting.
  </Step>
  <Step title="Attack and fingerprint">
    Use Batch Queries to test rate limit bypass scenarios. Run Engine Fingerprinting to identify the underlying GraphQL server. Bruteforce the schema if introspection is disabled.
  </Step>
</Steps>

<Note>
  InQL supports both Burp Suite Professional and Community editions. It requires Java 17 or later via the Montoya API.
</Note>
